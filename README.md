# GitHub Task Manager

A collaborative task management system integrated with GitHub, enabling public users to manage tasks through a modern web UI. Tasks are stored in a GitHub repository and synchronized automatically.

## ğŸŒŸ Features

- **Public Collaboration**: Anyone can create, update, and manage tasks through the web UI
- **GitHub Integration**: Tasks are stored in a GitHub repository
- **Automatic Field Population**: Task ID, creation date, and creator are automatically generated
- **Template-Based Validation**: All tasks follow a structured schema with required and optional fields
- **Status & Priority Tracking**: Comprehensive status and priority options
- **Dependencies & Subtasks**: Support for task dependencies and parent-child relationships
- **Critical Path Tracking**: Mark tasks as critical path for project planning

   ```bash
   npm test
   npm run test:validate
   ```

## ğŸ“‹ Task Schema

### Field Categories

| Category | Fields | Description |
|----------|--------|-------------|
| **Automatic** | `task_id`, `created_date`, `creator_id`, `completed_date` | Auto-generated by the system |
| **Required** | `task_name`, `description`, `start_date`, `end_date`, `priority`, `status`, `estimated_hours`, `category_name` | Must be provided by user |
| **Optional** | `progress_percentage`, `actual_hours`, `is_critical_path`, `tags`, `assigned_workers`, `parent_task_id`, `dependencies`, `comments`, `attachments` | User can optionally provide |

### Valid Status Values
- `Not Started`, `In Progress`, `On Hold`, `Blocked`, `Completed`, `Cancelled`, `Pending Review`

### Valid Priority Values
- `Low`, `Medium`, `High`, `Critical`

### Dependency Types
- `FS` (Finish-to-Start), `SS` (Start-to-Start), `FF` (Finish-to-Finish), `SF` (Start-to-Finish)
